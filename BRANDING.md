## .vya File Extension Claim

### Definition
A `.vya` file is a plain-text source code file written in the Vyākṛti programming language. The filename extension uniquely identifies Vyākṛti source code and distinguishes it from other programming languages.

### Canonical Use
The `.vya` extension is reserved for source code that:
1. Conforms to the official Vyākṛti language syntax and semantics
2. Uses the Devanagari script for keywords and identifiers (by convention)
3. Compiles and executes on the Vyākṛti compiler/VM

### Licensing
All `.vya` files created for the Vyākṛti project are subject to the **Apache License 2.0** (see LICENSE file).

When distributing `.vya` files:
- Include the full Apache License 2.0 text in any derivative work
- Retain copyright notices from the original `.vya` file
- If modified, clearly mark the changes
- Comply with all terms in Section 4 (Redistribution) of the Apache License

## Derivative Works & Extensions

### Using the .vya Extension in Forks/Derivatives
If you create a fork or derivative of Vyākṛti (e.g., a modified compiler, extended syntax, new VM backend), we respectfully request that you:

1. **Use a distinct file extension** to avoid confusion with the official Vyākṛti specification
   - Example: `.vya-ext`, `.vyax`, `.vya2`, or a completely new extension
2. **Clearly document the differences** from the official Vyākṛti language
3. **Update the documentation** to explain which syntax features are non-standard
4. **Credit the original project** in your README and LICENSE

This ensures that the `.vya` extension remains a reliable indicator of official Vyākṛti syntax and prevents user confusion.

### Example: Creating a Variant
If you fork Vyākṛti and add new features:
```
# DO use a distinct extension
myscript.vyax        ← Your dialect
myscript.vya-plus    ← Extended syntax
myscript.custom-vya  ← Custom variant

# DO NOT use .vya if your syntax diverges significantly
```
